Ordering execution of threads

Implement a function that takes three objects (instances of Thread or its subclasses). The function must start the passed objects as threads in a way that the order of their execution goes like this: t3, t2, t1. These threads print secret phrases to the standard output, their output must always be the same.

All given threads must be terminated before the implemented function is completed.

Otherwise, the testing system will give you some hints on throwing exceptions in the main thread, for example:

Exception in thread "main" java.lang.RuntimeException: All threads must be terminated before ending the implemented function


Memory limit: 256 MB
Time limit: 8 seconds
